# Phase 7 RC.7 reference-query runner

## Outcome

The frozen non-neural selected-prior reference now has a fail-closed future
execution runner and bounded score-artifact writer. This is implementation
evidence only. No scientific reference-bank record, validation record, final
case, neural checkpoint or calibration product was opened.

## Exact future gate

A later query authorization must bind all of:

- the terminal `lock_train_65k` decision and SHA-256;
- the development-only 12-result selected-architecture decision;
- the immutable corrected 65,536-system training publication;
- the selected primary rung's input standardizer and preparation artifact;
- one exact query parent and dataset manifest;
- immutable execution commit, wheel and environment;
- one query role, exact count and collision-free output root.

Validation is fixed at 6,144 systems. Final IID and balanced-tail reference
queries are fixed at 8,192 and 4,096 systems and additionally require the
separate final-evaluation unsealing gate. The implementation authorization is
rejected by this scientific gate before it resolves any publication path.

## Bounded execution

The corrected locked training rung is indexed once through the metadata-only
reader and selected-rung standardizer. Queries are group-disjoint and processed
one at a time. Each line contains deterministic KDE NLP, CRPS, central
marginal/joint coverage, interval widths and neighbor identity. The aggregate
summary records raw success counts, rates and Wilson 95% intervals by level,
lens family and EM cell.

The 4,096 deterministic posterior draws exist only for the current case and
are never persisted. The runner does not open query GW strain, access a neural
checkpoint, refit calibration, compute importance-sampling efficiency or call
the reference an exact likelihood or gold posterior. Small score, summary,
bank-identity and execution-manifest products use atomic writes and reject an
existing output identity.
